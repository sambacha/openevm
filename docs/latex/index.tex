 {\bfseries{ABI version 10}}

The EVMC is the low-\/level ABI between Ethereum Virtual Machines (EVMs) and Ethereum Clients. On the EVM-\/side it supports classic EVM1 and \href{https://github.com/ewasm/design}{\texttt{ ewasm}}. On the Client-\/side it defines the interface for EVM implementations to access Ethereum environment and state.\hypertarget{index_guides}{}\doxysection{Guides}\label{index_guides}

The EVMC project uses \href{https://semver.org}{\texttt{ Semantic Versioning}}. The version format is {\ttfamily MAJOR.\+MINOR.\+PATCH}.

The {\itshape MAJOR} version number is also referenced as the {\bfseries{EVMC ABI version}}. This ABI version is available to VM and Host implementations by \mbox{\hyperlink{group__EVMC_gga06fc87d81c62e9abb8790b6e5713c55ba1db9144f8af4e62865f0c003c07a02bc}{EVMC\+\_\+\+ABI\+\_\+\+VERSION}}. For example EVMC 3.\+2.\+1 would have ABI version 3 and therefore this project release can be referenced as EVMC ABIv3 or just EVMC 3. Every C ABI breaking change requires increasing the {\itshape MAJOR} version number.

The releases with {\itshape MINOR} version change allow adding new API features and modifying the language bindings API. Backward incompatible API changes are allowed but should be avoided if possible.

The releases with {\itshape PATCH} should only include bug fixes. Exceptionally, API changes are allowed when required to fix a broken feature.\hypertarget{index_modules}{}\doxysection{Modules}\label{index_modules}
